# 广西台风救援信息平台 · Typhoon Rescue System

面向广西台风/洪涝灾害的**移动端优先**公益救援信息平台。整合官方台风预报、分级救援电话、民间互助（求助/志愿/物资）、灾后重建与安全指南；并按 PRD 建设"超简化 AI 拍照求助 + 物资站点地图 + 罕见病患者"核心能力。

> 参考原型：香港大埔火灾救援协调系统。目标人群：受灾群众，**特别是方言沟通困难者与罕见病患者家庭**。

## ✨ 功能模块

| 模块 | 说明 | 后端 |
|---|---|---|
| 🏠 首页 | 台风态势横幅、紧急拨号、模块入口、近期灾情速览 | 无 |
| 🌀 台风预报 | 当前台风、近期影响广西的台风、预警信号、时间线、**官方数据源直达** | 无 |
| 📞 救援电话 | 全国主线 / 灾害专线 / 广西 14 市 / 民间救援组织，**分级核实标注**、一键拨号 | 无 |
| 🤝 求助互助 | **工单化求助墙**（待对接→对接中→已解决，认领去重、双向回写）、志愿者/救援队报名、物资捐赠（防骗提示） | Supabase |
| 📖 灾后指南 | 返家安全 / 用电燃气 / 饮水防疫 / 清淤 / 次生灾害 / 心理 / 重建理赔（含高危提示） | 无 |
| 🆘 拍照求助 | 拍照/粘贴群消息 + 引导式结构化表单 → 求助墙分派（AI 代理接入后自动预填，未接入用引导表单兜底） | 可选 AI |
| 📦 物资/安置 | 安置点/物资点/医疗点 + 一键导航（高德/腾讯 URI，零 key 零备案）+ 官方查询入口 | 无 |
| 🧬 罕见病 | 灾时援助热线 + 基金会 + 分病种（胰岛素/透析/供氧/癫痫/化疗）应急要点 + 应急包清单 | 无 |

> 拍照求助、物资/安置、罕见病 三个 PRD 核心模块收在「求助互助」页的横向二级 Tab，首页有醒目「拍照求助」大按钮直达。

**只读模块（预报/电话/指南/物资/罕见病）零后端，任何静态主机都能跑，中国大陆免翻墙可访问。**

## 🔒 数据核实纪律（救灾场景，号码错误会害人）

电话号码分级标注：`✓已核实`（权威一手/全区统一）· `官方媒体核实`（建议拨前再确认）· `待核实`（未收录，请拨 12345 兜底）。
凡无法从权威来源确认的号码一律不收录、不杜撰。灾情/伤亡数据以广西应急管理厅官方通报为准。

## 🛠 技术栈

- Vite + React 18 + TypeScript + Tailwind CSS（移动端优先，底部 Tab 导航）
- HashRouter（无需服务器重写，任意静态主机可部署）
- **PWA 离线可用**：Service Worker 预缓存，台风停电断网时仍可查看预报/电话/指南/罕见病；支持"添加到主屏幕"像 App 一样打开
- Supabase（互助模块后端，**未配置时自动降级为只读/演示**，站点其余部分不受影响）
- 内容数据为 TS 模块（`src/data/`），Git 版本管理、易更新

## 🚀 本地开发

```bash
npm install
cp .env.example .env.local   # 可选：填 Supabase 参数启用互助后端
npm run dev                  # http://localhost:5173
npm run build                # 产物在 dist/，可部署到任意静态主机
```

## ☁️ 部署（中国大陆免翻墙）

> 📖 完整步骤（前端静态托管 + 互助后端 + AI 代理）见 **[DEPLOY.md](DEPLOY.md)**。

构建产物是纯静态文件，推荐（按免翻墙优先级）：

1. **自有已备案域名 + 腾讯云 COS / 阿里云 OSS 静态托管**（大陆访问最快、最稳）。
2. **腾讯云 CloudBase（云开发）静态托管**（大陆原生，可同时承载后端云函数）。
3. Cloudflare Pages（多数情况大陆可访问，零备案起步快）。

> ⚠️ Supabase 的 `*.supabase.co` 在大陆可能不稳。若上互助后端，建议改用腾讯云 CloudBase / 阿里云等大陆可达的 BaaS，数据访问层（`src/lib/aid.ts`）已做成可替换。

## 🗄 互助后端建表

见 [`supabase/schema.sql`](supabase/schema.sql)。在 Supabase SQL Editor 执行后，把项目 URL 与 anon key 填入 `.env.local`。
