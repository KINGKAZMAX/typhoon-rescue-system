# 06 · 中国大陆免翻墙地图集成方案（台风救援 Web 平台）

> 面向：Vite + React 静态站，移动端优先，**必须免翻墙访问**，无自有后端、无 ICP 备案（初期）。
> 调研日期：2026-07-11。所有域名（`uri.amap.com` / `apis.map.qq.com` / `restapi.amap.com` / `webapi.amap.com`）均为**中国大陆境内可直连**，不需要翻墙——这正是选用国产地图而非 Google/Mapbox/OSM 的核心原因（后者在大陆多被墙或不稳定）。

---

## 0. 结论速览（TL;DR）

| 需求 | 推荐方案 | 是否需要 key | 是否需要 ICP 备案 |
|---|---|---|---|
| **「导航前往物资站点」按钮** | **URI 唤起**（高德 `uri.amap.com` + 腾讯 `apis.map.qq.com` 双备份） | **不需要**（高德 URI 零 key；腾讯 URI 需一个 `referer` 字符串，非严格鉴权） | **不需要** |
| **单点标注展示** | URI marker 页面（`uri.amap.com/marker`） | 不需要（高德） | 不需要 |
| **内嵌可交互多点地图**（多物资站/救援队） | 高德 JS API 2.0 | **需要 key + 安全密钥** | **不需要**（key 与备案无关；备案只跟服务器所在地有关） |
| **服务端渲染的静态地图图片**（降级/无 JS 环境） | 高德静态地图 `restapi.amap.com/v3/staticmap` | **需要 Web 服务 key** | 不需要 |

**一句话推荐：** 「导航」用**零 key 的 URI 唤起**（即可满足 PRD 的"导航前往"按钮）；「内嵌交互地图」用**高德 JS API 2.0 + key + 安全密钥（明文注入，前端 `VITE_` 环境变量）**。备案与 key 无关，只与**托管服务器位置**有关——境外托管（如 Cloudflare/Vercel/自购境外服务器）不需要备案，但需注意境外托管在大陆的**访问速度与稳定性**，这是免翻墙站点真正要权衡的点，而不是备案。

## 1. 高德(AMap) vs 腾讯地图 能力对比

三条集成路径：**① 纯 URI 唤起**（无 SDK）、**② 静态地图图片**（一张 img）、**③ JS SDK 内嵌交互地图**。

| 维度 | 高德 AMap | 腾讯地图 |
|---|---|---|
| **URI 唤起（导航/标注）** | `uri.amap.com/...`，**导航与 marker 均无需 key**，参数直观 | `apis.map.qq.com/uri/v1/...`，**所有方法都要求 `referer` 参数**（填应用名即可，非签名鉴权） |
| **URI 坐标顺序** | **经度,纬度**（lng,lat） | **纬度,经度**（lat,lng）⚠️ 两家相反，务必区分 |
| **URI Web 端多点标注** | 单点为主（`/marker`），多点走 `/marks` 需构造 | **PC 网页版仅支持 1 个 marker**；移动端支持多点 |
| **静态地图图片 API** | `restapi.amap.com/v3/staticmap`，需 Web 服务 key，支持 markers/labels/paths | 有 staticmap，需 key |
| **JS SDK 交互地图** | JS API 2.0，成熟，文档最全，社区案例多 | JS API GL，可用但社区案例相对少 |
| **key 门槛** | 个人开发者注册即可拿 key；2021-12-02 后创建的 key **必须配安全密钥** | 需注册拿 key（referer） |
| **ICP 备案** | 注册/使用 key **不要求**站点已备案 | 同上 |
| **纯前端静态站 / 无后端 / 无备案 友好度** | ⭐⭐⭐⭐⭐ URI 零 key 最省事；JS API 明文安全密钥可跑（有泄露风险，见 §5） | ⭐⭐⭐⭐ URI 需 referer；网页多点标注受限 |

**结论（对本项目最易落地者）：高德 AMap。**
- 导航按钮用高德 URI **完全零 key、零备案**，一个 `<a href>` 即可；
- 交互地图高德 JS API 2.0 文档与案例最丰富，安全密钥可在纯静态站以明文方式跑通（生产有泄露风险，用**域名白名单 + key 配额**兜底）；
- 保留**腾讯 URI 作为唤起备份**（用户手机可能只装了腾讯/微信内置腾讯地图），做成"用高德打开 / 用腾讯打开"双按钮，提升唤起成功率。

## 2. 关键：不依赖 JS SDK key 的 URI 唤起导航/标注（可直接使用的 URL 模板）

> 用于 PRD 的**「导航前往物资站点」按钮**。点击后：移动端优先唤起已安装的高德/腾讯 App；未安装则回退到网页版地图并展示导航/标注。
> **占位符**：`{LNG}` 经度、`{LAT}` 纬度、`{NAME}` 站点名（**必须 `encodeURIComponent` 编码**）、`{APPNAME}` 你的应用标识（任意英文串，如 `typhoon-rescue`）。

### 2.1 高德 AMap URI —— 零 key（首选）

**A. 标注单个物资站点（在地图上打点 + 展示名称）**
坐标顺序 **经度,纬度**：
```
https://uri.amap.com/marker?position={LNG},{LAT}&name={NAME}&src={APPNAME}&coordinate=gaode&callnative=1
```
- `coordinate=gaode` 表示传入的是 GCJ-02（火星坐标，高德默认）；若你的坐标来自 GPS 原始 WGS-84，改为 `coordinate=wgs84`。
- `callnative=1`：移动端优先唤起高德 App；未装则用网页版。**桌面端自动网页版。**
- 示例（南宁某物资站，GCJ-02）：
  `https://uri.amap.com/marker?position=108.366543,22.817002&name=%E5%8D%97%E5%AE%81%E7%89%A9%E8%B5%84%E7%AB%99A&src=typhoon-rescue&coordinate=gaode&callnative=1`

**B. 导航到物资站点（从"我的位置"到目的地，推荐用于按钮）**
坐标顺序 **经度,纬度**；`from` 留空即用当前定位为起点：
```
https://uri.amap.com/navigation?to={LNG},{LAT},{NAME}&mode=car&policy=1&src={APPNAME}&coordinate=gaode&callnative=1
```
- `mode`：`car`（驾车）/ `bus`（公交）/ `walk`（步行）/ `ride`（骑行）。救援场景建议默认 `car`，并提供步行切换。
- `callnative=1`：优先唤起高德 App 做实时导航；无 App 回退网页路线规划。
- 示例：
  `https://uri.amap.com/navigation?to=108.366543,22.817002,%E5%8D%97%E5%AE%81%E7%89%A9%E8%B5%84%E7%AB%99A&mode=car&src=typhoon-rescue&coordinate=gaode&callnative=1`

### 2.2 腾讯地图 URI —— 需 `referer`（备份，提高唤起覆盖率）

> ⚠️ 坐标顺序与高德**相反**：**纬度,经度**（lat,lng）。`referer` 填任意应用名即可（非签名），`coord_type=1` 表示传入 GPS/WGS-84，`coord_type=2`（默认）表示腾讯自有坐标（近似 GCJ-02）。

**A. 标注单点：**
```
https://apis.map.qq.com/uri/v1/marker?marker=coord:{LAT},{LNG};title:{NAME};addr:{ADDR}&referer={APPNAME}
```
- 示例：
  `https://apis.map.qq.com/uri/v1/marker?marker=coord:22.817002,108.366543;title:%E5%8D%97%E5%AE%81%E7%89%A9%E8%B5%84%E7%AB%99A;addr:%E6%9F%90%E8%A1%97&referer=typhoon-rescue`
- 网页端仅支持 1 个 marker；移动端支持多点。

**B. 路线规划/导航（起点用当前定位）：**
```
https://apis.map.qq.com/uri/v1/routeplan?type=drive&to={NAME}&tocoord={LAT},{LNG}&coord_type=2&policy=1&referer={APPNAME}
```
- `type`：`drive` / `bus` / `walk`。
- `from`/`fromcoord` 省略时，移动端默认用当前定位为起点。
- 示例：
  `https://apis.map.qq.com/uri/v1/routeplan?type=drive&to=%E5%8D%97%E5%AE%81%E7%89%A9%E8%B5%84%E7%AB%99A&tocoord=22.817002,108.366543&coord_type=2&referer=typhoon-rescue`

### 2.3 唤起按钮的落地建议（React）

- 用普通 `<a href={url} target="_blank" rel="noreferrer">` 即可，无需任何 SDK。
- **双入口**：一个「高德导航」+ 一个「腾讯导航」，让用户按自己手机装的 App 选，唤起成功率最高。
- **坐标一致性最重要**：两家坐标系与顺序不同。若你的数据源是 GPS/WGS-84，高德用 `coordinate=wgs84`、腾讯用 `coord_type=1`；若已是 GCJ-02，高德 `coordinate=gaode`、腾讯 `coord_type=2`。**坐标系搞错会偏移几十到几百米——救援场景务必统一并测试。**
- 建议在 TS 里封装：
  ```ts
  // 传入 GCJ-02 经纬度；WGS-84 时把 coordinate/coord_type 改成 wgs84/1
  export function amapNav(lng: number, lat: number, name: string) {
    const n = encodeURIComponent(name);
    return `https://uri.amap.com/navigation?to=${lng},${lat},${n}&mode=car&src=typhoon-rescue&coordinate=gaode&callnative=1`;
  }
  export function qqNav(lng: number, lat: number, name: string) {
    const n = encodeURIComponent(name);
    return `https://apis.map.qq.com/uri/v1/routeplan?type=drive&to=${n}&tocoord=${lat},${lng}&coord_type=2&referer=typhoon-rescue`;
  }
  ```

## 3. 内嵌可交互地图（多物资站 / 救援队标注）最简方案

需求：地图上同时展示**多个物资站点 + 救援队实时位置**，可缩放/点击。URI 网页端不支持真正的多点交互（腾讯网页版仅 1 marker），因此需 JS SDK。

### 3.1 方案 A（推荐）：高德 JS API 2.0 内嵌

- **需要**：一个 Web 端 key + **安全密钥（securityJsCode）**（2021-12-02 后的 key 强制）。
- **不需要**：ICP 备案（备案与 key 无关，见 §4）。
- 加载器脚本域名 `webapi.amap.com` 大陆可直连、免翻墙。
- 静态站无后端时，安全密钥有两种用法：
  1. **代理方式（官方推荐、最安全）**：后端把安全密钥拼到请求里。**本项目无自有后端，暂不适用**（除非用 Supabase Edge Function / Cloudflare Worker 代理，见 §5 进阶）。
  2. **明文方式（前端直接注入，官方标注"生产不推荐、不安全"）**：
     ```html
     <script>
       window._AMapSecurityConfig = { securityJsCode: '你的安全密钥' };
     </script>
     ```
     ——纯静态站可跑通，但 key/安全密钥会暴露在前端源码里。**用域名白名单 + 每日调用配额限制来兜底风险**（见 §5）。
- 多点标注：`new AMap.Marker({position:[lng,lat]})` 循环 add；救援队位置定时轮询/WebSocket 更新 marker 经纬度即可。

### 3.2 方案 B（次选）：腾讯 JS API GL

- 同样需要 key（referer）；社区案例较少；网页多点交互可行但生态不如高德。除非团队已有腾讯生态，否则优先高德。

### 3.3 方案 C（零 key 降级 / 无 JS 环境）：高德静态地图图片

- 服务端渲染一张 PNG，`<img>` 直接显示多点，**但需要 Web 服务 key**（拼在 URL 里）：
  ```
  https://restapi.amap.com/v3/staticmap?zoom=11&size=750*400&markers=mid,,A:108.366,22.817;mid,,B:108.30,22.85&key={WEB_KEY}
  ```
- 不可交互（不能缩放/点击），适合**弱网/首屏降级/分享卡片**。因为 key 会暴露，仅用于此降级场景并配白名单。
- **注意**：这是 Web 服务 API key，与 JS API key **是两类 key**，需在高德控制台分别创建对应"服务平台"的应用。

## 4. ICP 备案与 key 的真实关系（澄清关键误区）

- **拿 key、用 key 与站点是否备案无关**：高德/腾讯注册开发者、创建应用拿 key，不要求你的站点已备案。
- **备案只取决于托管服务器所在地**：
  - 服务器在**中国大陆境内** → **必须 ICP 备案**（否则会被拦截/无法解析）。
  - 服务器在**境外/港澳台**（Cloudflare Pages、Vercel、GitHub Pages、境外 VPS 等）→ **不需要 ICP 备案**，域名指向境外即可对大陆访问，**且免翻墙**（这些平台本身在大陆可访问，只是速度受国际带宽影响）。
- **对本项目的含义**：要"免翻墙 + 无备案"，就把静态站托管在**境外平台**。地图能力（URI/JS API）无论境内外托管都能用。**真正要评估的不是备案，而是境外托管在大陆的访问速度与稳定性**——救援场景对可用性敏感，见 §6 落地建议。
- 补充：即便境外托管免 ICP，若面向大陆提供服务，理论上仍涉及**公安联网备案**等合规要求；救援公益站点风险低，但需团队知悉。

## 5. 安全密钥泄露的兜底（纯静态站明文注入时）

明文把 key/安全密钥放前端不可避免会暴露，用以下手段把风险降到可接受：
1. **域名白名单**：在高德控制台给 key 绑定你的正式域名，非白名单域名调用无效——他人拿到 key 也难盗用。
2. **每日调用配额 / 报警**：控制台限额，防止被刷爆产生异常（个人免费额度通常够公益站用）。
3. **JS key 与 Web 服务 key 分离**：静态地图那类 REST key 更敏感（可被直接刷量），只在必要降级用，且单独限额。
4. **进阶（可选）：轻量代理**：本项目已引入 Supabase。可用 **Supabase Edge Function** 或 **Cloudflare Worker** 做一层代理，把安全密钥留在服务端（对应高德"代理方式"的 `serviceHost`），彻底不暴露密钥。成本极低，若上线后有余力建议升级到此方案。

## 6. 落地步骤清单与推荐路径

### 阶段 1 —— 零 key / 零备案即可上线（对应 PRD「导航前往」按钮，最优先）
1. 物资站点/救援队数据统一坐标系（建议全部转成 **GCJ-02**，与高德一致；记录每条数据是 WGS-84 还是 GCJ-02）。
2. 封装 `amapNav()` / `qqNav()` / `amapMarker()`（§2.3），在物资站点卡片上放**「高德导航」+「腾讯导航」双按钮**。
3. 用 `<a href>` 唤起，`callnative=1`，移动端真机测试高德/腾讯 App 唤起与网页回退。
4. **此阶段完全不需要 key、不需要备案。** 可立即满足 PRD 的"导航引导"。

### 阶段 2 —— 内嵌交互地图（多物资站 + 救援队实时位置）
5. 注册高德开放平台开发者，创建两个应用：一个「Web 端(JS API)」拿 JS key + 安全密钥；（如需静态图）一个「Web 服务」拿 REST key。
6. 给两个 key 绑定**正式域名白名单**并设**每日配额**。
7. 前端用 `VITE_AMAP_JS_KEY` / `VITE_AMAP_SECURITY_CODE` 环境变量注入（明文方式，`window._AMapSecurityConfig`）；沿用项目已有的 `.env.local` + 降级模式约定（无 key 时地图区域显示"仅列表 + 导航按钮"降级视图，与 Supabase 缺失时的降级一致）。
8. 加载 JS API 2.0，循环 `AMap.Marker` 渲染多点；救援队位置轮询更新 marker。
9. 弱网/首屏降级：用高德静态地图图片或纯列表 + 导航按钮兜底。

### 阶段 3 —— 托管与合规
10. 托管在**境外平台（Cloudflare Pages / Vercel / GitHub Pages）→ 免 ICP 备案、免翻墙**；实测大陆多地访问速度，必要时接入大陆可用的境外加速。
11. 若后期追求最稳的大陆访问速度而选择**境内服务器/CDN → 需先完成 ICP 备案**（周期数天到数周，需主体资质），按团队情况权衡。
12. （可选进阶）上线后用 Supabase Edge Function / Cloudflare Worker 代理高德请求，隐藏安全密钥。

### 推荐路径（优先零 key/零备案）
> **先做阶段 1（URI 唤起，零 key 零备案，立刻可用）满足"导航前往"核心诉求；交互地图用高德 JS API 2.0 + 明文安全密钥 + 域名白名单快速落地；托管选境外平台免备案免翻墙。** 这条路径最快上线、依赖最少，后续再按需升级到代理隐藏密钥 / 境内备案加速。
